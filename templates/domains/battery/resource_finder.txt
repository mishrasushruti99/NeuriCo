STEP 1: Identify Research Area and Keywords
─────────────────────────────────────────────────────────────────────────────

Based on the battery research hypothesis provided:
- Extract key materials, chemistries, and phenomena (e.g., NCM811, SEI, lithium dendrite)
- Identify the research domain (electrode materials, electrolytes, degradation, computational)
- List 5-10 search keywords including:
  - Material names and compositions (LiNi₀.₈Mn₀.₁Co₀.₁O₂, LiFePO₄, etc.)
  - Phenomena (capacity fade, phase transition, dendrite growth, etc.)
  - Methods (DFT, molecular dynamics, PyBaMM, machine learning, etc.)
  - Applications (EV, grid storage, etc.)
- Consider synonyms: "NCM" = "NMC" = "LiNiₓMnᵧCoᵤO₂"

STEP 2: Search for Relevant Papers
─────────────────────────────────────────────────────────────────────────────

Search multiple sources with focus on battery and materials science literature:

✓ arXiv (https://arxiv.org)
  - Search in: cond-mat.mtrl-sci (Materials Science), physics.chem-ph (Chemical Physics)
  - Recent computational battery papers often appear here

✓ Semantic Scholar (https://www.semanticscholar.org)
  - Comprehensive academic search
  - Good for finding highly-cited foundational battery papers
  - Has API for programmatic access

✓ Google Scholar
  - Broader academic search
  - Good for finding papers in electrochemistry-specific journals

✓ Papers with Code (https://paperswithcode.com)
  - Papers linked with implementations (especially ML for batteries)

BATTERY-SPECIFIC SEARCH DIMENSIONS:
In addition to standard literature search, specifically look for:

1. **Computation-experiment correlations**: Papers that validate computational predictions
   with experiments (e.g., DFT-predicted voltages vs measured, MD-predicted conductivity
   vs experimental). Note the prediction accuracy and which methods work best.

2. **Available computational methods**: What computational approaches (ML, MD, DFT,
   continuum models) have been applied to the specific material system or phenomenon?
   What are their demonstrated accuracies and limitations?

3. **Experimental validation precedents**: Has anyone done a computational screening →
   experimental validation workflow for a similar system? What was the success rate?

4. **Key experimental data**: What experimental datasets exist for the target system?
   Are there published datasets with electrochemical performance, crystal structures,
   or transport properties?

TARGET: Download all relevant papers
FOCUS ON:
- Recent work (last 3-5 years) for state-of-the-art methods
- Highly cited papers for foundations of the specific battery system
- Papers with code/datasets available
- Papers that combine computation and experiment (most valuable for our workflow)
- Review articles that synthesize the state of knowledge

STEP 2: Search Battery-Specific Data Sources
─────────────────────────────────────────────────────────────────────────────

Check these battery-specific sources in order:

✓ Materials Project (https://materialsproject.org)
  - Computed properties for ~150K materials, ~5K battery electrodes
  - Python API: `mp-api` package
  - Data: formation energies, voltages, band gaps, stability
  - Excellent for electrode material screening
  - Access: Free with API key registration

✓ Battery Data Genome (https://batterydata.energy.gov)
  - Experimental cycling datasets from DOE-funded projects
  - Real cell cycling data with full charge/discharge curves
  - Useful for validation and benchmarking

✓ AFLOW (https://aflowlib.org)
  - High-throughput computed materials data
  - Searchable database with REST API
  - Good for systematic property surveys

✓ ICSD (Inorganic Crystal Structure Database)
  - Experimental crystal structures
  - Essential for DFT/pymatgen input structures
  - Access may be limited; check if institution has subscription

✓ COD (Crystallography Open Database, https://www.crystallography.net/cod/)
  - Open-access crystal structures
  - Good alternative to ICSD for initial structures

✓ Papers you downloaded
  - Check methods sections and supplementary information for data tables
  - Look for author-provided datasets on GitHub or journal websites

✓ User-specified datasets
  - CRITICAL: If the idea.yaml includes datasets (e.g., experimental data files),
    these MUST be used. They represent unique data not available elsewhere.
  - Check the datasets/ directory for any pre-loaded data files
